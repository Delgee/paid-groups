================================================================================
PAYMENT INTEGRATION DETAILS
================================================================================

GATEWAY: QPay Mongolia
ENVIRONMENT: Sandbox & Production ready
FLOW:
  1. User initiates payment → Backend creates PaymentTransaction
  2. Backend generates QPay invoice → Returns payment link
  3. User completes payment on QPay checkout
  4. QPay calls webhook: GET /v1/webhooks/qpay/{transactionId}
  5. Backend polls QPay API to verify payment
  6. Payment confirmed → BullMQ queues membership job
  7. Membership created → Auto-add to group
  8. User notified via Telegram DM

SECURITY:
  ✅ HMAC-SHA256 webhook verification
  ✅ Idempotency keys (unique qpay_invoice_id)
  ✅ 3-retry exponential backoff (1s, 2s, 4s)
  ✅ Webhook event deduplication
  ✅ Complete audit trail
  ✅ Payment state machine

FEATURES:
  • Auto-add users to Telegram groups on payment
  • Auto-remove on expiration
  • Telegram notifications (success/failure)
  • Invoice tracking
  • Transaction status tracking (pending/completed/failed/refunded)

SECURITY ARCHITECTURE
================================================================================

AUTHENTICATION:
  • JWT tokens with 15-minute expiry
  • Refresh tokens with 7-day expiry
  • HTTP-only cookies (secure storage)
  • Token refresh flow automatic

AUTHORIZATION:
  • Role-based access control (RBAC)
  • Roles: SUPER_ADMIN > OWNER > ADMIN > MODERATOR > MEMBER
  • Per-endpoint role validation with @UseGuards
  • Audit logging for sensitive operations

MULTI-TENANCY:
  • Row-Level Security (RLS) enforced at DB level
  • TenantInterceptor sets app.current_tenant context
  • PostgreSQL session variable prevents cross-tenant leaks
  • Defense-in-depth: both app and DB level checks

DATA PROTECTION:
  • Passwords hashed with bcrypt (10 salt rounds)
  • Bot tokens masked in API responses
  • Sensitive fields excluded from JSON serialization
  • Encrypted webhook secrets in environment
  • Correlation IDs for request tracing

PAYMENT SECURITY:
  • HMAC-SHA256 signature verification
  • Idempotency checks (UNIQUE constraint)
  • PCI compliance via QPay partner
  • Event deduplication
  • Complete audit trail
